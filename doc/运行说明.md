## 一、数据库初始化

**若需要重置数据库，按顺序执行以下脚本：**

```powershell

# 2. 注入种子数据（200条商品 + 20个测试用户）
cmd /c "mysql -h 121.41.92.121 -u group_user -p12345678 swapcampus_db < backend/sql/seed.sql"

# 3. 建立积分游戏化相关表（sign_record / task_config / user_task）
cmd /c "mysql -h 121.41.92.121 -u group_user -p12345678 swapcampus_db < backend/sql/credit-gamification.sql"

# 4. 修复旧商品封面图 URL（把 demo.jpg 换成 picsum 占位图）
cmd /c "mysql -h 121.41.92.121 -u group_user -p12345678 swapcampus_db < backend/sql/fix-cover-url.sql"
```

## 二、MinIO 对象存储

MinIO 用于存储商品图片和聊天图片，**需要在本地启动**。

**1. 下载 MinIO（Windows）**

从 https://min.io/download 下载 `minio.exe`，放到任意目录。

**2. 启动 MinIO**

```powershell
# 在 minio.exe 所在目录执行
.\minio.exe server D:\minio-data --console-address ":9001"
```

启动后：
- API 地址：`http://localhost:9000`
- 控制台：`http://localhost:9001`（默认账号/密码：`minioadmin` / `minioadmin`）

**3. 创建 Bucket**

访问控制台 → 创建名为 `swapcampus` 的 Bucket → 将访问策略设置为 **Public**（或 `Read-Only`）。

> **注意：** 后端启动时会自动检测并创建 Bucket，并设置公开读策略，所以 MinIO 正常运行即可，无需手动创建。

## 三、启动后端

```powershell
# 在项目根目录执行
cd backend

# 方式一：Maven 直接运行（推荐开发时使用）
mvn -pl swapcampus-app spring-boot:run

# 方式二：先打包再运行
mvn clean package -DskipTests
java -jar swapcampus-app/target/swapcampus-app-0.0.1-SNAPSHOT.jar
```

后端启动后监听 `http://localhost:8080`。

**验证后端是否正常：**
```powershell
curl http://localhost:8080/actuator/health
# 返回 {"status":"UP"} 即正常
```

## 四、启动前端

```powershell
# 进入前端目录
cd frontend

# 首次运行需安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端启动后访问 `http://localhost:5173`。

**Vite 开发服务器代理配置（已内置，无需修改）：**

| 前缀 | 代理目标 | 说明 |
|------|---------|------|
| `/api` | `http://localhost:8080` | 后端 REST 接口 |
| `/ws` | `ws://localhost:8080` | WebSocket 聊天 |
| `/minio` | `http://localhost:9000` | MinIO 图片访问 |

## 八、常见问题

| 现象 | 原因 | 解决 |
|------|------|------|
| 图片全部显示"暂无图片" | MinIO 未启动 | 启动 MinIO，访问 `localhost:9000` 验证 |
| 登录后 token 失效 | JWT 过期（默认24h） | 重新登录 |
| 上传图片失败 | Bucket 不存在或无权限 | 重启后端（自动创建 Bucket）或手动在控制台创建 |
| WebSocket 聊天连不上 | 后端未启动或端口被占用 | 检查后端 8080 端口 |
| 后端启动报数据库连接失败 | 网络不通或服务器宕机 | 检查能否 ping 通 `121.41.92.121` |
