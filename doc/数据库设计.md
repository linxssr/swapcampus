## 二、索引设计 

`-- 1.` 用户表：加速学号登录查询 `CREATE UNIQUE INDEX idx_student_id ON `user`(student_id);` 

`-- 2.` 商品表：加速按发布者 `/` 分类 `/` 审核状态查询 `CREATE INDEX idx_item_user_id ON `item`(user_id); CREATE INDEX idx_item_category_id ON `item`(category_id); CREATE INDEX idx_item_audit_status ON `item`(audit_status);` 

`-- 3.` 订单表：加速按买家 `/` 卖家 `/` 订单状态查询 `CREATE INDEX idx_order_buyer_id ON `orders`(buyer_id); CREATE INDEX idx_order_seller_id ON `orders`(seller_id); CREATE INDEX idx_order_status ON `orders`(order_status);` 

`-- 4.` 聊天表：加速一对一聊天记录查询 

```
CREATE INDEX idx_chat_from_to ON `chat_message`(from_uid, to_uid);
```

`-- 5.` 举报表：加速按处理管理员查询 

```
CREATE INDEX idx_report_handle_admin ON `report`(handle_admin_id);
```

## 四、新增表（2026年6月）

### 11. 商品浏览历史表（item_browse_history）

用于记录用户的商品浏览行为，作为个性化推荐的数据来源。

| 字段名 | 数据类型 | 约束 | 注释 |
|---|---|---|---|
| history_id | bigint | PK、自增 | 记录ID |
| user_id | bigint | FK、NOT NULL | 用户ID |
| item_id | bigint | FK、NOT NULL | 商品ID |
| category_id | bigint | NOT NULL | 商品分类ID（冗余，便于统计） |
| browse_time | datetime | NOT NULL | 最近浏览时间 |

**约束：** `UNIQUE KEY uk_user_item (user_id, item_id)`，同一用户对同一商品只保留一条记录，重复浏览时更新 `browse_time`。

**可执行 SQL：**

```sql
-- 11. 商品浏览历史表（item_browse_history）
CREATE TABLE IF NOT EXISTS `item_browse_history` (
  history_id  bigint       AUTO_INCREMENT PRIMARY KEY COMMENT '记录ID',
  user_id     bigint       NOT NULL COMMENT '用户ID',
  item_id     bigint       NOT NULL COMMENT '商品ID',
  category_id bigint       NOT NULL COMMENT '商品分类ID（冗余）',
  browse_time datetime     NOT NULL COMMENT '最近浏览时间',
  UNIQUE KEY  uk_user_item (user_id, item_id),
  FOREIGN KEY (user_id)  REFERENCES `user`(user_id)  ON DELETE CASCADE,
  FOREIGN KEY (item_id)  REFERENCES `item`(item_id)  ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='商品浏览历史表';

CREATE INDEX idx_browse_user_id   ON `item_browse_history`(user_id);
CREATE INDEX idx_browse_user_time ON `item_browse_history`(user_id, browse_time DESC);
```
